# Join Space Overview

This section covers all the available methods for managing spaces, including joining, leaving, starting, stopping, requesting microphone access, toggling user audio in spaces, and other actions.

  <title>Join Space Overview | Push Spaces | Push Documentation</title>
</head>

## Prerequisites

- You need to [initialize user](/docs/spaces/build/initialize-user/ "Initialize User for Push Spaces") before initializing video.
- You also need to declare `data`, `setData` which are essentially a state/variable to hold space-related data with a callback function to react to changes in space state.

## Initialize Space API

Push Space API abstracts away connection and encryption methods to enable seamless audio spaces between the connected users.

<Tabs className="codetabs" groupId="code-examples">
  <TabItem value="js" attributes={{className: "codetab js"}} default>

```typescript
  // await userAlice.space.initialize({ onChange: setData, spaceId: spaceId });
  const userAliceSpace = await userAlice.space.initialize({
    onChange: setData,
    spaceId: spaceId, // Space ID that you want to join/start
  });

  </TabItem>
</Tabs>
```

---

### Initialize Space API parameters

| Property     | Type       | Sub Type | Description                                                   |
| ------------ | ---------- | -------- | ------------------------------------------------------------- |
| _`onChange`_ | `function` | -        | Callback function to react to changes in space state.         |
| _`spaceId`_  | `string`   | -        | Space ID of the given Push Space that you want to join/start. |

> Note: Parameters _`in this style`_ are mandatory.

<Details summary="Expected Response">
200 OK

</Details>

## Start a Space

This API is used to start a space. It can be utilized by the space creator to start their space. Once the space is started, listeners will be able to join.

<Tabs className="codetabs" groupId="code-examples">
  <TabItem value="js" attributes={{className: "codetab js"}} default>

```typescript

await userAliceSpace.start();
```

  </TabItem>
</Tabs>

<Details summary="Expected response">

200 OK

</Details>

## Stop/End a Space

This API is used to stop/end a space. It can be utilized by the space creator to end their space. Once the space is ended, no listeners will be able to join there after.

## Accept Mic Request

This API is used to accept a mic request from a listener in the given space. Space creator or admin can accept the listener's request to speak by calling this method.

<Tabs className="codetabs" groupId="code-examples">
  <TabItem value="js" attributes={{className: "codetab js"}} default>

```typescript
// await userAliceSpace.acceptMicRequest({address, signal});
await userAliceSpace.acceptMicRequest({
  address: address,
  signal: signal,
});
```

  </TabItem>
</Tabs>
---

### Accept Mic Request API parameters

| Property    | Type     | Sub Type | Description                                                  |
| ----------- | -------- | -------- | ------------------------------------------------------------ |
| _`address`_ | `string` | -        | Address of the listener that you want to accept the request. |
| _`signal`_  | `string` | -        | Signal data received in the stream.                          |

<Details summary="Expected response">

200 OK

</Details>

## Toggle Audio

This API is used to toggle local audio in the given space. Space creator or admin can toggle their by calling this method.

<Tabs className="codetabs" groupId="code-examples">
  <TabItem value="js" attributes={{className: "codetab js"}} default>

```typescript
// await userAliceSpace.config({audio: boolean})
await userAliceSpace.config({
  audio: boolean,
});
```

  </TabItem>
</Tabs>
---

### Toggle Audio parameters

| Property  | Type      | Sub Type | Description                                                                                |
| --------- | --------- | -------- | ------------------------------------------------------------------------------------------ |
| _`audio`_ | `boolean` | -        | Boolean value to toggle the audio. `TRUE` to enable audio and `FALSE` to mute their audio. |

> Note: Parameters _`in this style`_ are mandatory.

<Details summary="Expected response">

200 OK

</Details>
